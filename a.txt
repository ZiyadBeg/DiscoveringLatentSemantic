Here's a review of your solution.</em></p><p>It's important to use meaningful, descriptive names for your program elements. It's impossible to guess what a class named "Improved" will do, and what a variable named "A" might be. Try to come up with better names.</p><p>Instead of putting some code in a <code>main</code> method, setting some hardcoded values, doing some logic and printing a result, it would be better to create a method with a single purpose, with a good name, input parameters and return value.</p><p>In the loop, you don't really need the loop index variable. In cases like this, it's strongly recommended to use an enhanced for-each loop instead.</p><p>The formatting of the code is also unusual, and doesn't follow common Java conventions.</p><p>Something like this would be better:</p><pre><code>public class ArrayUtils {    public static int findMaxDifference(int[] arr) {        assert arr.length &gt; 0;        int min = arr[0];        int max = arr[0];        for (int value : arr) {            if (min &gt; value) {                min = value;            }            if (max &lt; value) {                max = value;            }        }        return max - min;    }}</code></pre><p>Note that the <code>assert</code> keyword serves mostly as documentation, in production code it has no effect, typically only enabled during unit test runners or debuggers.</p><blockquote>  <p>The only problem I got now is counting the runtime. I think for the pseudocode, it runs in O(n^2) because of the 2 for loops.
